# Changelog

Все важные изменения в проекте Fishertools будут документированы в этом файле.

## [0.5.5] - 2026-02-23

### Changed

- Unified network download response/progress models
- Lazy-loaded top-level `fishertools` submodules to reduce import coupling
- Cleaned up decorators docs/messages and removed unreachable branches in safe file helpers

### Tests

- Moved REPL tests out of package runtime path into `tests/test_learn_repl`
- Reduced OneDrive-related Hypothesis/temp-directory issues in test configuration

## [0.5.3] - 2026-02-19

### Added

- Unified learning CLI improvements:
  - `fishertools learn list`
  - richer `fishertools learn --help` examples
- API behavior modes docs and guarantees docs
- Plugin hooks for custom formatters, error patterns, and i18n providers
- Release flow documentation and version consistency guard tests
- CI typing baseline (mypy) job

### Changed

- Single source of truth for package version (`fishertools/_version.py`)
- Improved file/path and logger edge-case handling

### Tests

- Added CLI smoke, plugin hook, performance smoke, and release consistency tests
- Added security and diagnostics-focused regression suites

## [0.5.2.2] - 2026-02-13

    - `concatenation` - конкатенация с информацией о типах

- **Решение:** 
  - Создан `ExplanationLoader` с использованием `importlib.resources` для надежной загрузки
  - Обновлены MANIFEST.in и setup.py для включения JSON файлов
  - Добавлены fallback механизмы для разных версий Python
  - Четкие сообщения об ошибках при отсутствии файлов

- **Проблема:** `validate_number("string", 0, 100)` выдавал непонятную ошибку сравнения операторов
- **Решение:**
  - Обернуты внутренние сравнения типов в try-catch блоки
  - ValidationError теперь содержит фактический и ожидаемый типы
  - Специальная обработка для None значений
  - Сообщения: "Expected number, got str" вместо технических ошибок Python

- **Новая функция:** `explain_error()` для образовательных объяснений ошибок валидации

- Примеры исправления для распространенных ошибок
- Fallback объяснения для неизвестных паттернов
- Прогрессивно детальные объяснения для повторяющихся ошибок

#### 🎨 Улучшено форматирование строк (safe_format)
- **Новая функция:** Настраиваемое поведение для отсутствующих плейсхолдеров
- **PlaceholderBehavior enum:**
  - `PRESERVE` - оставляет оригинальные плейсхолдеры: `{name}`
