## [0.4.6] - 2026-02-02

### ⚡ Performance & Async Release

**Значительные улучшения производительности и поддержка асинхронного программирования**

#### 🚀 Новые возможности

##### Асинхронная поддержка
- **AsyncSimpleLogger** - асинхронная версия SimpleLogger для async/await приложений
  - Не блокирует event loop
  - Async-safe с `asyncio.Lock`
  - Те же методы: `info()`, `warning()`, `error()`
  - Использует `asyncio.to_thread()` для неблокирующих операций

- **Async Safe Utilities** - асинхронные версии безопасных утилит
  - `async_safe_read_file()` - асинхронное чтение файла
  - `async_safe_write_file()` - асинхронная запись файла
  - `async_safe_file_exists()` - асинхронная проверка существования
  - `async_safe_get_file_size()` - асинхронное получение размера
  - `async_safe_list_files()` - асинхронный список файлов

##### Потокобезопасность
- **SimpleLogger** теперь потокобезопасен
  - Добавлен `threading.Lock` для защиты от race conditions
  - Безопасное использование в многопоточных приложениях
  - Автоматическая синхронизация записи в файл

##### Кэширование
- **PatternLoader** использует LRU cache
  - Добавлен `@functools.lru_cache(maxsize=1)` декоратор
  - Паттерны загружаются только один раз
  - Ускорение повторных вызовов в ~10,000 раз

##### PEP 561 Support
- Добавлен `py.typed` marker file
  - Полная поддержка type checkers (mypy, pyright)
  - Type hints теперь доступны для внешних проектов
  - Обновлен `pyproject.toml` для включения файла

##### Future Annotations
- Добавлен `from __future__ import annotations` во все модули
  - Улучшена производительность импорта
  - Поддержка отложенной оценки аннотаций типов
  - Совместимость с Python 3.8+

#### 🏗️ Структурные изменения

##### Современная сборка
- **Удален setup.py** - используется только `pyproject.toml`
  - Соответствие PEP 517/518
  - Упрощенная структура проекта
  - Единый источник конфигурации

#### 📚 Документация
- Добавлено руководство по асинхронным возможностям (`ASYNC_GUIDE.md`)
- Подробная документация улучшений (`IMPROVEMENTS_v0.4.6.md`)
- Примеры использования async функций
- Сравнение производительности sync vs async

#### ✅ Тестирование
- Добавлены тесты для всех новых возможностей
- Тесты потокобезопасности SimpleLogger
- Тесты кэширования PatternLoader
- Тесты асинхронных функций

## [0.4.5] - 2026-02-02

### 🐛 Critical Bug Fixes

**Исправлены критические ошибки в fishertools v0.4.4.2**

#### ✅ Исправлен модуль обучения (Learning Module)
- **Проблема:** FileNotFoundError при вызове `explain("lists")` - файл explanations.json не включался в пакет
- **Решение:** 
  - Создан `ExplanationLoader` с использованием `importlib.resources` для надежной загрузки
  - Обновлены MANIFEST.in и setup.py для включения JSON файлов
  - Добавлены fallback механизмы для разных версий Python
  - Четкие сообщения об ошибках при отсутствии файлов

#### 🔧 Улучшены сообщения об ошибках валидации
- **Проблема:** `validate_number("string", 0, 100)` выдавал непонятную ошибку сравнения операторов
- **Решение:**
  - Обернуты внутренние сравнения типов в try-catch блоки
  - ValidationError теперь содержит фактический и ожидаемый типы
  - Специальная обработка для None значений
  - Сообщения: "Expected number, got str" вместо технических ошибок Python

#### 📖 Добавлены контекстные объяснения ошибок
- **Новая функция:** `explain_error()` для образовательных объяснений ошибок валидации
- Система сопоставления паттернов ошибок с объяснениями
- Примеры исправления для распространенных ошибок
- Fallback объяснения для неизвестных паттернов
- Прогрессивно детальные объяснения для повторяющихся ошибок

#### 🎨 Улучшено форматирование строк (safe_format)
- **Новая функция:** Настраиваемое поведение для отсутствующих плейсхолдеров
- **PlaceholderBehavior enum:**
  - `PRESERVE` - оставляет оригинальные плейсхолдеры: `{name}`
  - `MISSING` - заменяет на: `[MISSING: name]` (по умолчанию)
  - `EMPTY` - заменяет пустой строкой
- Четкий API для конфигурации поведения
- Обратная совместимость с существующим кодом

#### ➕ Добавлена функция safe_average()
- **Новая функция:** Безопасное вычисление среднего с защитой от ошибок
- Возвращает указанное значение по умолчанию для пустых списков
- Автоматически фильтрует нечисловые значения
- Примеры:
  - `safe_average([1, 2, 3])` → `2.0`
  - `safe_average([], default=0)` → `0`
  - `safe_average([1, "text", 3])` → `2.0` (фильтрует "text")

#### 🛡️ Комплексная обработка ошибок
- Образовательные сообщения об ошибках для всех функций fishertools
- Объяснения для начинающих для распространенных ошибок Python
